# WeData 设计系统

## 1. 视觉主题与氛围

WeData 的界面是一份**数据报告的气质**——干净的页面、克制的排版、精确到小数点的数值，以及说完就走的专业态度。如同一张精心排版的分析报表：标题黑，正文灰，重点蓝，其余一切退到背景中。这不是作为美学潮流的极简主义，而是作为数据工程纪律的极简主义。每一个像素都服务于数据、AI 的推理过程，以及你的下一步操作。界面本身则消隐不见。

视觉根基是一组冷调蓝灰色板，基于 TDesign 的 BlueGray 色阶构建。默认画布色是 `#F2F4F8`——不是纯白，而是带有微妙蓝灰底色的浅灰。这一抹蓝灰避免了纯白的冷冽无菌感，同时保持了企业级数据工作所需的清爽利落。在这层画布之上，白色卡片和面板作为主要内容承载面浮现，形成含蓄的层次纵深，无需依赖厚重的阴影。

字体系统以中文为先，极度克制。苹方 (PingFang SC) 承载整个 UI，仅使用两个工作字重：Regular (400) 用于正文，Medium (500) 用于强调。Semibold (600) 只在展示级标题中出现。这种「双字重纪律」意味着层次感主要通过我们称之为**「墨色四阶」**的体系构建——一套四级 rgba 黑色透明度 (0.9 / 0.7 / 0.5 / 0.3)——而非粗细对比。如同中国水墨画中浓、重、淡、清的四种墨色层次：0.9 是标题和正文的浓墨，0.7 是辅助信息的重墨，0.5 是占位符和元数据的淡墨，0.3 是禁用态的清墨。最终效果是：乍看平静统一，细读层次分明。

形态语言由**「胶囊统一律」**定义。所有交互控件——按钮、搜索框、标签页、AI 入口——共享 100px 的圆角，形成柔软、亲近的胶囊形态。如同 Notion 的 "whisper border"（耳语边框）定义了 Notion 的边框哲学，WeData 的胶囊统一律定义了它的形态哲学：不存在方角按钮和圆角按钮的选择——一切交互都是胶囊。这种激进的统一是 WeData 3.0 最鲜明的视觉签名。

色彩几乎不出现在界面装饰中。整个界面在灰阶中运行，仅有一个功能蓝 (`#1664FF` / `#0052D9`) 专门用于交互元素。品牌青色 (`#00C8D6`) 只在品牌触点出现：logo、AI 入口边框、二级 Tab 的激活下划线。AI 入口按钮是全界面唯一的视觉华彩——多层渐变叠加青色与橙色角光，传达「此处有智能」的信号，却不破坏周围的平静。

**核心特征：**
- 苹方 (PingFang SC) 双字重克制 (400/500)，通过**墨色四阶**（rgba 四级透明度）而非字重建立层次
- 冷调蓝灰画布 (`#F2F4F8`) + 白色卡片面——不是纯白，也不是暖白
- 唯一的功能色：蓝色 (`#1664FF`) 用于交互；青色 (`#00C8D6`) 仅用于品牌标识
- **胶囊统一律** (100px 圆角) 贯穿所有交互控件——不存在方角按钮
- **墨色四阶**：`rgba(0,0,0, 0.9/0.7/0.5/0.3)` 对应浓/重/淡/清四级文字层次
- 对话页面近乎零装饰——灰阶 + 一个蓝色强调色即为全部调色板
- 880px 居中内容置于宽视口中——大量留白作为注意力聚焦漏斗
- AI 入口按钮的多层渐变 (青色/橙色角光) 是唯一的装饰性时刻
- 海獭吉祥物插画作为 AI Agent 头像——温度通过角色注入，而非界面装饰
- 毛玻璃效果仅用于输入框聚焦态：conic-gradient 旋转边框 + 弥散模糊光晕

## 2. 设计原则

1. **克制即信号** — 色彩越少，每个色彩的意义越强。一个蓝色 = 可点击。一个青色 = 品牌。暖色 = 决策。没有灰色地带。
2. **层次靠透明度，不靠粗细** — 中文排版中字重跳变比拉丁文更突兀。墨色四阶 (0.9 → 0.7 → 0.5 → 0.3) 比从 Regular 跳到 Bold 更自然，如同水墨画中浓、重、淡、清的四种墨色层次。
3. **胶囊即交互** — 100px 圆角 = "你可以操作我"。≤24px 圆角 = "我是容器"。没有中间态。圆角的二分法消除了"这个元素是按钮还是卡片"的视觉歧义。
4. **信息容器静默，展示容器发声** — 任务卡用 1.5px 边框界定（信息不需要浮起来），Agent 名片卡用弥散阴影浮起（品牌需要被看见）。同一界面内，不同内容类型享有不同的纵深待遇。
5. **AI 是唯一华彩** — 整个界面在灰阶中运行，AI 入口按钮是唯一允许装饰的时刻。所有"智能感"集中释放于此——渐变、光晕、品牌青边框。这种极度克制的释放反而让 AI 入口成为全界面的视觉锚点。

> 以上五条原则从全文已有设计决策提炼而来，是所有具体规格的判断依据。当遇到文档未覆盖的边缘场景时，按此原则推导即可。

### 排版原则
- **中文优先的视觉适配**：对话正文 16px 配合 28px (1.75×) 行高，专为中文多段落长文阅读优化。标准 UI 正文同为 16px 但使用 24px (1.50×) 行高以适应更紧凑的界面场景。CJK 字形天然较宽，需要比拉丁文字更大的行间呼吸空间。
- **双字重纪律**：组件中仅出现 Regular (400) 和 Medium (500)。Semibold (600) 保留给标题。UI 系统中不存在 Bold (700)（仅 Hack 等宽徽章使用）。层次感来自透明度和字号，而非字重的剧烈跳变。
- **零字间距为默认**：整个系统的 letter-spacing 均为 0。CJK 文字自带字符间距，施加负字距会破坏中文的可读性。唯一例外是 TDesign 的大写 overline token。
- **对话三级经济**：聊天页面仅使用 16px / 14px / 12px 三个字号。在同一字号内，通过字重 (400 → 500) 和透明度 (0.3 → 0.9) 构建层次，而非增加额外的字号级别。
- **TDesign 字重映射说明**：TDesign 将 `--tea-font-weight-medium` 定义为 `600`，但苹方的 "Medium" 字型实际渲染为 500。在 Figma 组件中，`PingFang SC: Medium` = 500，`PingFang SC: Semibold` = 600。始终以实际 CSS `font-weight` 数值为准，而非字型名称。

- 柔滑: `cubic-bezier(0.2, 0, 0, 1)` — 面板展开等丝滑过渡

- 微交互: 100ms — 悬停背景色过渡、图标切换

## 6. 布局原则

### 间距系统
- 基础单位: 4px（TDesign `--tea-space-base`）
- 阶梯: 0, 4, 8, 12, 16, 20, 24, 28, 32, 36, 40, 44, 48, 52, 56, 60, 64px（17 级）
- 节奏规律: 元素间 16–24px，卡片/容器间 24px，紧凑组件内部 4–8px，面板间距 20px

- 网格系统: 基于 flex 的面板布局，非严格 CSS Grid
- 最大内容宽度: 880px（对话页） / 满宽（工作台页）
- 面板间距: 20px
- 对齐方式: 对话页居中单列，工作台面板分割

| 面板 | 宽度 |
|------|------|

### 留白哲学
- **大量居中即聚焦装置**：880px 内容置于宽视口中，利用率约 50%。周围的留白不是空的——它是一个注意力漏斗，将视线牢牢锁定在对话上。每个元素都存在于一片宁静场域的中心。当用户在 1696px 的屏幕上与 AI 对话时，左右各 400px 的留白等同于一堵隐形墙，将注意力封闭在中央通道里。
- **透明度即留白替代（墨色四阶的空间功能）**：WeData 不依赖空间分隔来区分文字层级，而是使用透明度层 (0.9 → 0.7 → 0.5 → 0.3)。视觉上的「呼吸空间」来自淡化而非像素间距。以任务进度卡为例：头部标题 (0.9)、进度计数 (0.5)、待执行步骤 (0.3) 三者垂直排列，仅靠墨色浓淡就建立了清晰的三级层次——无需增加任何额外间距或分割线。这使得密集的信息展示不会产生视觉噪音。
- **卡片分组优于线条分隔**：留白是首要的结构元素，边框是次要的。绝大多数板块分隔靠背景色切换（`#F2F4F8` → `#FFFFFF` → `#F7F8FB`）和间距实现。显式分割线仅用于表格内部和面板之间。这与 Notion 的「耳语边框」策略异曲同工——都是让结构感从留白中自然浮现，而非用线条强行划定。

## 7. 深度与层级

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 平面 (Level 0) | 无阴影 | 页面背景、文字块、平面内容 |
| XS | `0 1px 4px rgba(0,0,0,0.05)` | 最小浮起（TDesign） |
| 描边按钮 (Level 1a) | `0 2px 4px -2px rgba(0,0,0,0.12)` | 描边按钮、搜索框、AI 入口 |
| 填充按钮 (Level 1b) | `0 2px 4px -2px rgba(0,0,0,0.20)` | 主要深色按钮——比描边略强 |
| SM / 默认 | `0 2px 8px -1px rgba(0,0,0,0.05), 0 2px 4px 2px rgba(0,0,0,0.05)` | 下拉菜单、浮层（TDesign） |
| Tab 激活态 | `0 3px 6px -3px rgba(0,0,0,0.08), 0 6px 12px -6px rgba(0,0,0,0.04)` | 激活胶囊 Tab 浮起 |
| Tab 容器 | `inset 0 2px 2px rgba(0,0,0,0.03)` | 胶囊 Tab 容器内凹 |
| MD | `0 8px 24px -4px rgba(0,0,0,0.1), 0 8px 12px -8px rgba(0,0,0,0.05)` | 弹窗、浮起面板（TDesign） |
| 卡片 | `0 4px 16px rgba(0,0,0,0.06)` | Agent 名片卡、特色卡片 |
| LG | `0 24px 48px -12px rgba(0,0,0,0.15)` | 展示级大幅浮起（TDesign） |

**阴影哲学**：WeData 的阴影体系有两条设计轴线。

第一条是**负扩展紧贴法**。按钮阴影 `0 2px 4px -2px` 中，`-2px` 的扩展值使阴影向内收缩，最终比元素本身还窄。2px 模糊在 -2px 扩展下，产生一条纸张厚度的边缘阴影——几乎不可见却结构性地存在，如同一张纸放在桌面上的投影。Tab 激活态的 `-3px` 和 `-6px` 扩展同理：每一层都紧紧贴合元素轮廓，而非向外弥散。这不是多数 SaaS 产品那种环境光晕，而是精确的、工程师式的阴影。

第二条是**有阴影 vs 无阴影的功能性区分**。任务卡——信息容器——完全放弃阴影，依赖 1.5px 边框实现界定，因为在数据平台中，结构清晰优先于装饰纵深。Agent 名片卡——品牌展示——才使用 `0 4px 16px rgba(0,0,0,0.06)` 的弥散环境光，因为展示类内容可以承受一点戏剧性。这种「信息容器无阴影、展示容器有阴影」的二分法，是 WeData 深度系统的核心决策。

**毛玻璃效果**：`backdrop-filter: blur(20px) saturate(60%)` + 微妙渐变边框——仅用于聊天输入框的激活态。这是界面唯一允许自己华丽一下的时刻，传达「AI 正在聆听」的信号。

### 装饰性纵深
- 面色调变化（`#F2F4F8` → `#F7F8FB` → `#FFFFFF`）在不使用阴影的情况下营造微妙的层次纵深
- 卡片头部使用 `#F7F8FB` 背景，在白色卡片内制造类似内凹的层次
- 胶囊 Tab 容器的 `inset` 阴影创造了一条凹陷轨道，激活的 Tab 从中「浮起」

## 8. 应做与禁忌

### 应做
- 将所有文字色设为 `rgba(0,0,0,α)`——**墨色四阶** (0.9/0.7/0.5/0.3) 是首要层次工具，而非字重或字号
- 为每一个按钮、搜索框、Tab、交互控件施加胶囊圆角 (100px)——**胶囊统一律**不允许例外
- 保持对话页面近乎零装饰——灰阶 + 一个蓝色强调色即为全部，让数据和 AI 回复成为主角
- 将暖沙色 (`#FFF8F0`、`#F5EDE0`) 限定在确认卡和产出物场景——这是冷调系统中唯一允许的暖色出口
- 维持描边按钮的渐变方向从上到下 (top → bottom)——模拟自然顶光，保持全局光源一致性
- 将品牌青色 (`#00C8D6`) 限定在品牌触点：logo、AI 入口边框、二级 Tab 激活下划线——稀缺创造辨识
- 将功能蓝 (`#1664FF` / `#0052D9`) 专用于交互元素：链接、选中态、主 CTA——蓝色 = 可点击
- 遵守 4px 基础单位——所有间距必须是 4 的倍数，杜绝 5px、7px 等非对齐值
- 使用 ✦（星光）作为 AI 活动的通用指示符——无论加载中、执行中还是完成态，✦ 是 AI 的视觉身份
- 通过圆角区分卡片类型：16px 用于任务卡（信息容器），24px 用于 Agent 名片卡（品牌展示）

### 禁忌
- 禁止用纯黑 `#000000` 做正文——纯黑与 `#F2F4F8` 蓝灰背景的对比过于刺眼，始终使用 `rgba(0,0,0,0.9)` 作为最深阅读色。纯黑仅保留给胶囊 Tab 激活态和当前步骤强调，因为这些场景需要「跳出墨色四阶」的极端醒目
- 禁止将语义色（绿/红/橙）用于界面装饰——WeData 的界面温度是冷调蓝灰，彩色仅在数据展示、状态图标和警告中才具有语义价值；滥用会破坏「报告感」的基调
- 禁止使用 24px 到 100px 之间的圆角——这个空档是刻意的：≤24px 的弧度用于静态容器，100px 的胶囊用于交互控件。中间值会模糊「容器」与「按钮」的视觉区分
- 禁止添加弹跳、弹簧或装饰性动画——数据平台的用户在高认知负荷下工作，非功能性动效是干扰。所有过渡遵循 `cubic-bezier(0.4, 0, 0.2, 1)` 缓动，像呼吸一样自然
- 禁止在单个视图中使用超过 3 个字号——对话页 16/14/12，工作台 14/12/10。层次不足时用透明度（墨色四阶）而非新字号来解决
- 禁止表格行斑马纹——WeData 用统一白底 + 微妙底部边框 (`1px #E9ECF1`) 构建表格节奏；斑马纹在蓝灰色系中会产生色温冲突，且破坏「干净面板」的整体视觉印象
- 禁止在悬停时使用 scale/transform——缩放变形暗示物理弹性，而 WeData 的控件是平面的、精确的。悬停反馈仅通过背景变暗 + 阴影升级实现，保持工程感
- 禁止引入超出已定义范围的额外强调色——色彩预算已用尽：青色（品牌标识）+ 蓝色（功能交互）+ Agent 专属色（角色区分）。新增色彩必然与现有信息架构冲突
- 禁止用 `#0052D9` 做下划线 Tab——这是 WeData 对 TDesign 的重要覆盖：用品牌青 `#00C8D6` 替代默认蓝，在导航层级注入品牌辨识度
- 禁止给任务卡加阴影——任务卡用 1.5px 边框界定，因为它是信息容器而非展示元素；阴影保留给 Agent 名片卡等品牌展示场景

## 9. 响应式行为

### 视口策略
WeData 当前为**固定桌面端设计**，主要设计目标为宽屏桌面视口 (~1696px)。最小支持宽度约 1280px（侧边导航 224px + 导航栏 48px + 文件树 280px + 内容区 + 属性面板 360px 的最小可用布局）。设计系统中未定义移动端或平板断点。

| 场景 | 视口 | 布局 |
|------|------|------|
| 对话 | ≥1280px 桌面 | 侧边栏 (224px) + 居中内容 (最大 880px) |
| 工作台 | ≥1280px 桌面 | 导航栏 (48px) + 文件树 (280px) + 内容 (flex) + 属性面板 (360px) |
| 折叠导航 | ≥1024px 桌面 | 侧边栏 (72px) + 扩展内容 |

### 触控目标
| 元素 | 尺寸 | 说明 |
|------|------|------|
| 按钮 大 | 高 44px | 主 CTA、搜索框 |
| 按钮 小 | 高 32px | 次要操作、紧凑场景 |
| 胶囊 Tab 项 | 高 40px | 一级视图切换 |
| 菜单项 | 高 36px | 侧边导航 |
| 树节点 | 高 32px | 文件树、设置列表 |
| 导航栏图标 | 32px 容器 | 二级导航 |

| 开关 | 36×20px | 开/关控件 |
| 分页按钮 | 24×24px | 页码导航 |

### 折叠策略（未来）
- 导航: 224px 侧边栏 → 72px 折叠 → 移动端隐藏 + 汉堡菜单
- 属性面板: 360px → 小屏幕覆盖弹窗
- 文件树: 280px → 覆盖抽屉
- 内容宽度: 880px 最大 → 流式缩窄至最小
- 胶囊 Tab: 保持水平布局，溢出滚动

### 迭代指南
1. 文字层次靠**墨色四阶**（透明度）而非字号——`rgba(0,0,0, 0.9/0.7/0.5/0.3)` 对应浓/重/淡/清四级，场景映射：0.9 正文和标题 → 0.7 辅助信息和表头 → 0.5 占位符和元数据 → 0.3 禁用和待执行
2. **胶囊统一律**：所有交互元素都是 100px 圆角——按钮、搜索框、Tab、AI 入口，无一例外
3. 间距始终为 4px 的倍数：4/8/12/16/20/24/28/32
4. 按钮系统只有两种模式：填充（深色，rgba 黑）和描边（渐变灰）。没有幽灵按钮，没有纯文字按钮
5. AI 存在感通过以下元素传达：✦ 星光图标、青色边框 (`#BCEFEC`)、多层渐变背景、海獭吉祥物
6. 阴影在按钮和 Tab 上使用负扩展 (-2px, -3px) 实现紧贴边缘的阴影；卡片上使用零扩展实现弥散环境光
7. 暖色 (`#FFF8F0`、`#F5EDE0`) 专门保留给确认/决策 UI 和产出物图标——绝不用于通用背景
8. 对话页面最多使用 3 个字号 (16/14/12)——抵制添加第 4 个的冲动
9. 品牌青 (`#00C8D6`) 只出现在品牌触点：logo、AI 入口、下划线 Tab。其他所有交互用功能蓝 (`#1664FF` / `#0052D9`)
10. 任务卡有边框、无阴影。Agent 名片卡有阴影、有边框。两者的处理方式不可混用
