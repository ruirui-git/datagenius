# AI 时代体验设计原则

## Overview
来自乔布斯、原研哉、尼尔森、深泽直人、唐·诺曼五位设计大师圆桌讨论的7条核心原则，构成AI体验设计的哲学框架。

## Learning Objectives
The viewer will understand:
1. AI体验设计的7条核心原则及其层级关系
2. "无意识为默认，有意识为通道"的核心框架
3. 如何判断AI在何时应"安静完成"何时应"留空邀请"

---

## Section 1: 核心信念

**Key Concept**: 一切原则围绕一个信念展开——AI应服务于人的强大，而非AI的炫技

**Content**:
- "AI时代的体验设计，不是让AI更聪明，而是让AI更安静；不是让用户惊叹AI的能力，而是让用户惊叹自己的能力。"

## Section 2: 哲学层（原则一 & 二）

### 原则一：无意识为默认，有意识为通道

**Key Concept**: AI交互应默认无意识，但有意识的通道始终可接入

**Content**:
- 无意识是默认状态，有意识是备用通道
- 触发是设计者决定的，接入是用户决定的——把主动权交还给用户
- 环境应保留足够的"粗糙度"和感觉入口，让人随时可以回到有意识

**Visual Element**:
- Type: 双层结构图
- Subject: 外层透明/虚线=无意识默认，内层实线=有意识通道
- Treatment: 层叠嵌套，通道始终可见但非强制

**Text Labels**:
- Headline: "原则一：无意识为默认，有意识为通道"
- Label 1: "无意识 = 默认"
- Label 2: "有意识 = 通道"
- Label 3: "主动权在用户"

### 原则二：安静与空

**Key Concept**: AI有两种隐退方式——安静是态度，空是结构

**Content**:
- 安静：AI把事做完但不打扰你（态度）
- 空：AI把事做到未完成，邀请你进入（结构）
- 重量感是留空的唯一判据——当这个时刻对用户有重量时，留空；当它只是流程中的一个节点时，安静完成
- 同一功能在不同语境下需要不同处理（日常邮件安静完成，辞职信留空让人进入）
- "空"必须有暗示——茫然的空是虚无，有暗示的空是邀请

**Visual Element**:
- Type: 对比图
- Subject: 左侧"安静"（闭合的容器），右侧"空"（开口的容器）
- Treatment: 安静用实心淡色，空用空心留白，中间标注"重量感判据"

**Text Labels**:
- Headline: "原则二：安静与空"
- Left: "安静：做完不打扰"
- Right: "空：未完成，邀请进入"
- Divider: "重量感 = 留空的判据"

## Section 3: 方法层（原则三 & 四）

### 原则三：场景替代界面，容错也是无意识

**Key Concept**: 交互从场景开始而非界面，容错和正常操作一样直觉

**Content**:
- 可供性不再来自物体形状，而来自用户行为的惯性和环境上下文
- 场景替代界面是对的，但场景必须容错
- 容错本身也应该是无意识的——修正动作和正常操作一样直觉
- 容错不需要"报错界面"或"纠正流程"，需要的是自然的滑回
- 可见性的重新定义：默认隐藏，按需显现，永远可达

**Visual Element**:
- Type: 场景示意图
- Subject: 左=空白输入框（划叉），右=情境感知界面（勾选）
- Treatment: 下方标注容错的"自然滑回"

**Text Labels**:
- Headline: "原则三：场景替代界面"
- Label 1: "默认隐藏"
- Label 2: "按需显现"
- Label 3: "永远可达"
- Label 4: "容错 = 无意识修正"

### 原则四：AI是新的合作物种，不是人

**Key Concept**: 拟人化造成不切实际的期望，AI应是一种新的协作对象

**Content**:
- 赋予AI人脸、名字、性格，暗示它拥有人的理解力和共情力——但它没有
- AI应被设计为一种新的"合作物种"：有自己的能力和局限，有自己的行为模式
- 用户学习与它协作，就像与不同性格的同事协作——不是当人，也不是当工具

**Visual Element**:
- Type: 三分对比图
- Subject: 工具 ← → 合作物种 ← → 人
- Treatment: 中间"合作物种"高亮，两端灰掉

**Text Labels**:
- Headline: "原则四：新的合作物种"
- Left: "工具"
- Center: "合作物种"
- Right: "人"
- Sub: "不是人，也不是工具"

## Section 4: 保障层（原则五、六、七）

### 原则五：适当信任

**Key Concept**: 设计目标不是让用户信任AI，而是保持恰当程度的信任

**Content**:
- AI太聪明 → 过度依赖；AI坦诚局限 → 完全拒绝
- 恰到好处的困惑让用户知道"这里有不理解的东西"，维持反思层的活跃度
- 完全的透明滋生盲目信任，完全的困惑引发拒绝——中间地带才是安全的

**Visual Element**:
- Type: 信任光谱/天平
- Subject: 左端=盲目信任，右端=完全拒绝，中间=适当信任
- Treatment: 中间区域高亮标记为安全区

**Text Labels**:
- Headline: "原则五：适当信任"
- Left: "过度依赖"
- Center: "适当信任"
- Right: "完全拒绝"

### 原则六：行为先手权与意识在场

**Key Concept**: AI时代必须守护人的两条底线

**Content**:
- 行为先手权：动作从用户自身出发，而非被AI替代——保证人没有被替代
- 意识在场：用户可以随时介入自己的行为流程——保证人没有消失在流畅中
- AI替你做决定（替代）是危险的，AI响应你的行为（延伸）才是正确的
- 无意识不等于"不属于你"——走路时你不会意识到脚掌如何着地，但走路属于你

**Visual Element**:
- Type: 双柱/双轨图
- Subject: 左柱=行为先手权，右柱=意识在场
- Treatment: 两柱并列支撑，标注"延伸 ≠ 替代"

**Text Labels**:
- Headline: "原则六：行为先手权 × 意识在场"
- Left: "先手权：人没有被替代"
- Right: "意识在场：人没有消失在流畅中"
- Bottom: "延伸 ≠ 替代"

### 原则七：可用性是底线

**Key Concept**: 十大可用性原则在AI时代更加紧迫，但需重新诠释

**Content**:
- 系统状态可见性 → AI必须让用户知道正在发生什么，信息可验证、可追溯
- 与现实世界匹配 → 语言流畅制造"理解幻觉"——语言流畅 ≠ 任务完成
- 一致性与标准 → 各家AI交互模式不统一造成"负面迁移"代价极高
- 灵活性与效率 → 对话框不应是唯一交互方式，AI的工作流需要可复用

**Visual Element**:
- Type: 四项指标卡
- Subject: 功能可发现率、意图表达成功率、结果验证效率、错误恢复成本
- Treatment: 每项指标独立卡片，配微型图标

**Text Labels**:
- Headline: "原则七：可用性是底线"
- Label 1: "功能可发现率"
- Label 2: "意图表达成功率"
- Label 3: "结果验证效率"
- Label 4: "错误恢复成本"

## Section 5: 反模式

**Key Concept**: 当前AI产品最常见的7个设计错误

**Content**:
1. 急于表现 — 害怕沉默
2. 要求主动对话 — "我可以帮你什么？"
3. 空白输入框 — 放弃观察责任
4. 拟人化陷阱 — 暗示不存在的理解力
5. 不可发现 — 用户必须靠猜
6. 不可验证 — 无法判断输出正确性
7. 不一致 — 同样的输入不同的输出

**Visual Element**:
- Type: 清单/叉号列表
- Subject: 7项反模式，每项配简短描述
- Treatment: 红色叉号标记，紧凑排列

**Text Labels**:
- Headline: "7大反模式"
- Items: "急于表现", "要求主动对话", "空白输入框", "拟人化陷阱", "不可发现", "不可验证", "不一致"

## Section 6: 检验标准

**Key Concept**: 三个问题快速自检

**Content**:
1. 用户是否需要学习才能使用？— 如果需要，设计失败
2. 用户是否感到自己更强大了？— 如果只是AI更强大，设计失败
3. 用户是否能随时介入？— 如果AI的流程无法被用户打断，设计失败

**Visual Element**:
- Type: 三个问题/检验门
- Subject: 三道检验关卡
- Treatment: 问号图标+判断箭头

**Text Labels**:
- Headline: "三问自检"
- Q1: "需要学习？→ 失败"
- Q2: "用户更强？→ 通过"
- Q3: "随时介入？→ 通过"
