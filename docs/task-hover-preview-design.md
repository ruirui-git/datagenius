# 最近任务 Hover 预览浮层 — 设计文档

## 1. 核心问题：用户靠什么识别一个历史任务？

侧栏的任务标题通常被截断，用户看到的可能只是一行省略文字。用户需要一种方式**不点进去就能判断这是不是我要找的那个任务**。

### 1.1 展示原始提示词 还是 AI 总结？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **A. 用户原始提示词** | "自己说过的话"记忆锚点最强，0.5秒确认 | 可能很长、口语化；多轮对话后意图可能已偏移 |
| **B. AI 生成摘要** | 简洁规范，能反映最终实际做了什么 | 多一层认知转换，用户不确定"这是不是我那个" |
| **C. 原始提示词 + AI 补充** | 兼顾两者 | 占空间，可能冗余 |

**结论：选 A — 原始提示词优先。**

大数据场景下，用户的提示词本身就是需求描述（"统计近7天各渠道用户支付金额"），已经足够具体。人对自己说过的话有天然锚点，不需要再翻译一层。

**边界处理**：原始提示词 < 15 字时（如"帮我看看数据"），在下方追加一行 AI 补充说明。

---

## 2. 不同状态下用户的核心关注

| 状态 | 用户心声 | 浮层需要回答的问题 |
|------|---------|-------------------|
| **执行中** | "跑到哪了？要不要等？" | 当前步骤 + 进度 |
| **等待中** | "卡在哪了？需要我做什么？" | 等待原因 |
| **已完成** | "产出了什么？能直接用吗？" | 产物列表 + 关键结果 |
| **失败** | "为什么挂了？能重试吗？" | 失败原因 |

## 3. 浮层统一结构

```
┌──────────────────────────────────────────┐
│  ● 状态标签 [· 进度]             相对时间  │  ← 第一区：状态 + 时间
│                                          │
│  "用户的原始提示词，最多展示三行            │  ← 第二区：识别锚点
│   文字，超出部分截断..."                   │
│                                          │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │
│                                          │
│  产物/进度/等待原因/失败原因               │  ← 第三区：按状态变化
│  （具体内容因任务类型和状态而异）           │
│                                          │
└──────────────────────────────────────────┘
```

## 5. 关键设计决策总结

### 5.1 为什么是原始提示词而不是 AI 总结？

实际验证——用户想找"上次做的渠道支付统计"：

**方案 A（原始提示词）**：看到"统计近 7 天各渠道用户支付金额…" → 0.5秒确认。
**方案 B（AI 总结）**：看到"渠道支付金额 7 日趋势统计" → "嗯？是这个吗？好像是…" → 多一层犹豫。

原始提示词是用户自己的语言，记忆回溯最快。

### 5.2 多轮对话怎么处理？

当对话经历多轮迭代，最终产物和原始提问有差距时：
- **原始提示词照常展示**（用于识别"这是哪个任务"）
- **追加一行**"共 N 轮对话（补充说明最终方向偏移）"
- 这样用户既能快速认出，又不会被误导

### 5.3 不同类型的"产物区"差异

| 任务类型 | 产物区展示重点 | 为什么 |
|----------|---------------|--------|
| 临时取数 | SQL 文件名 + 结果行数/字段数 | 用户想复用 SQL 或导出结果 |
| 指标开发 | 指标表名 + 口径公式 + 核心数值 | 用户要回溯口径、引用指标 |
| 数据接入 | 源→目标映射 + 数据量 + 调度状态 | 用户关心数据有没有到位 |
| 报表看板 | 看板名 + 图表清单 | 用户要找到那个看板在哪 |
| 调度编排 | 工作流名 + 节点数 + 最近执行状态 | 用户关心有没有在正常跑 |
| 质量/血缘 | 结论摘要 + 风险发现 | 产出是分析结论不是数据实体 |

### 5.4 状态的精细化表达

不是简单的"执行中/等待/完成"，而是带上下文的状态：

| 原始状态 | 精细化表达 | 场景 |
|----------|-----------|------|
| 执行中 | "执行中 · SQL 运行中" | 取数 |
| 执行中 | "执行中 · 步骤 3/5" | 多步任务 |
| 执行中 | "同步中 · 64%" | 数据同步 |
| 等待中 | "排队中" | 资源排队 |
| 等待中 | "等待确认" | 需要用户决策 |
| 已完成 | "已完成" | 正常完成 |
| 已完成 | "已完成 ⚠ 最近执行异常" | 调度任务配好了但运行出错 |
| 失败 | "失败" / "同步失败" | 具体说明是哪种失败 |
